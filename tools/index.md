# Инструменты и рекомендации

Рекомендации по использованию инструментов в процессе разработки Unity-игры. Глубина и частота использования зависят от выбранного [режима разработки](../MODE_CHOICE.md).

---

| Инструмент | Назначение | Файл |
|------------|------------|------|
| **Unity MCP** | Управление Unity Editor из Cursor: сцена, объекты, скрипты, скриншоты, консоль | [unity-mcp.md](unity-mcp.md) |
| **ComfyUI** | Генерация визуальных ассетов: спрайты, текстуры, фоны, иконки | [comfyui.md](comfyui.md) |

## Использование инструментов по режимам

| Инструмент | Прототип | Стандартный | Быстрый | Профи |
|------------|----------|-------------|---------|-------|
| **Unity MCP** | Минимум: refresh + скриншот в конце | На каждую фичу: состояние, скриншот, консоль | Refresh по ходу; скриншот в конце | Полный: на задачу и фичу, batch_execute |
| **ComfyUI** | Не нужен (плейсхолдеры) | По желанию: спрайты для финалки | По желанию | Полный: все ассеты через генерацию |
| **Figma** | Не нужен | По желанию: быстрый макет UI | Не нужен | Рекомендуется: макеты всех экранов |
| **Unity Editor** | Минимально: сцена + Play | Стандартно: сцена, инспектор, Play | Стандартно | Полный: профайлер, анализ, настройки |
| **UI Builder** | UI только через UI Builder (UXML/USS, UIDocument) | То же | То же | То же |
| **C# / Код** | Хардкод + SO | Умеренный: компоненты + SO | Компоненты + SO | Архитектура + SO + тесты + интерфейсы |

---

## Правило выбора

> **Инструмент нужен, только если он ускоряет результат в текущем режиме.** Не подключай Figma для прототипа и не пропускай автотесты в Профи.

Подробные рекомендации — в файлах каждого инструмента.
